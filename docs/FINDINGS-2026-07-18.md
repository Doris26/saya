# FINDINGS.md — AI Voice Input MVP 去风险 spike 实测结论

> 版本: v1.0 | 日期: 2026-07-18 | 状态: VERIFIED(本机实测 + adversarial verify)
> 机器: macOS 26.3.1 arm64 · Command Line Tools only(**无 Xcode.app / 无 xcodebuild**)· SDK MacOSX15.5 · Swift 6.1.2
> 一句话总纲: PLAN 的技术主线全部成立,但 **§5「Xcode 工程」整章要换成 SwiftPM + bundle.sh**,**§2.5 的 data-protection keychain 在 ad-hoc 签名下直接 -34018 不可用**,**§6 分发链缺 $99/年账号无法执行**;另有一条 PLAN 未写的 correctness bug(转写会随机吐繁体中文)。

规则声明:本文每条结论都带实测命令+输出。被 adversarial verify 判为「未能独立复现」或原 spike 明确「未证明」的,一律标 **未验证**,不写「应该可以」。SPIKE 4(AVFoundation 录音)结果未随本轮交付,本文不覆盖。

## 2. PLAN.md 需要修订的地方(具体到章节号)

### §5 全章 + §5.2「Xcode 工程创建」——**必须重写为 SwiftPM**(最大改动)
- **原 plan 说**:§5.1 目录树含 `AIVoiceInput.xcodeproj`;§5.2 步骤 1「Xcode → New Project → macOS App」、步骤 2「Signing & Capabilities」、§6「Xcode Archive → Distribute App」。整条构建/分发链假设装了 Xcode。
- **实测是**:本机 `xcodebuild -version` → `tool 'xcodebuild' requires Xcode ... is a command line tools instance`;`/Applications/Xcode.app` 不存在。**但** `swift build -c release` 完整跑通 MenuBarExtra + Carbon 热键 + 手工 .app 打包 + `codesign --force --sign -`,全部 CLT 完成(SPIKE 1/2 证据)。
- **应改成**:构建产物改为 `Package.swift`(swift-tools-version 6.0/6.1)+ 约 40 行 `bundle.sh`(见 §3)。删除 `.xcodeproj`、`xcodebuild`、「New Project」「Archive → Distribute」等一切 Xcode 步骤。目录树把 `AIVoiceInput.xcodeproj` 换成 `Package.swift`。§5.2 步骤 6 的 test target 改用 `swift test`。
- **附带**:§5.2 步骤 1 deployment target `macOS 14.0` **可用**(`.macOS(.v14)` OK)。但符号上限是 `.v15`(SDK 15.5),**`.macOS(.v26)` 直接编译失败** `reference to member 'v26' cannot be resolved`。任何 macOS 26 / SDK 16-only API 不可达,需 plan 级替代。

### §2.5 Keychain 存取——**去掉 `kSecUseDataProtectionKeychain=true`**
- **原 plan 说**:代码里 `kSecUseDataProtectionKeychain as String: true`,注释「macOS 上强烈建议」;§2.5 结尾把 ad-hoc 读不到旧值描述为「开发期属正常现象」。
- **实测是**:ad-hoc 签名下 `kSecUseDataProtectionKeychain=true` → `ADD OSStatus=-34018 (A required entitlement isn't present.)`,**硬失败,不是弹窗、不是「读不到旧值」**。想加 `keychain-access-groups` entitlement「修」它 → AMFI 直接 `EXIT=137 (SIGKILL)`,main() 都进不去。对照实验:非 restricted 的 `allow-jit` entitlement 能跑,证明触发点是 entitlement 的 **restricted 属性**、不是「带 entitlement」本身。
- **应改成**:Developer ID 之前的阶段,**省略该 flag,用 legacy 文件钥匙串**——实测 `ADD/READ/UPDATE/DELETE 全 OSStatus=0`、CJK 字节级还原、零 GUI 弹窗。SecItem API/service 串/`kSecAttrAccessibleAfterFirstUnlock` 全不变,只删一行 flag。买了 Developer Program 后再决定是否加回 DP 钥匙串。

### §2.1 全局热键——**补 Swift 6 约束,确认「无需权限」**
- **原 plan 说**:§2.1 表格「RegisterEventHotKey 无需任何权限」;坑里给 `InstallEventHandler` target 用 `GetEventDispatcherTarget()`;修饰键 cmdKey=256/optionKey=2048。
- **实测是**:「无需权限」**CONFIRMED**——`trusted=false` 全程 4/4 收到热键、零弹窗(SPIKE 2)。spike 用的 target 是 `GetApplicationEventTarget()`,`RegisterEventHotKey err=0`。修饰键映射实测:NSEvent `[.control,.option]` raw 786432 → carbon 6144 = 0x1800 = `controlKey|optionKey`,round-trip 相等。
- **应改成**:补一条 **Swift 6 language mode 约束**——`@convention(c)` 的 `EventHandlerUPP` 回调及其共享状态(`hotKeyRef`/`fireCount`)在 `main.swift` 顶层是隐式 `@MainActor`,从 nonisolated 上下文引用会编译报错 `main actor-isolated let ... can not be referenced from a nonisolated context`;**必须声明 `nonisolated(unsafe)`**。verify agent 额外发现 injector 侧的顶层 `CGEventSource` 同样中招,规则:任何被 nonisolated / `@convention(c)` 函数触碰的顶层引用型状态都要 `nonisolated(unsafe)`。另注:必须是真 .app bundle + 活着的 NSApplication runloop(`setActivationPolicy(.accessory)` + `LSUIElement=true`),裸 SwiftPM 二进制收不到热键。

### §2.2 文字注入——**chunk≈20 标 未验证;剪贴板还原降级为「有损」**
- **原 plan 说**:§2.2「单个事件只可靠携带 ~20 个 UTF-16 code unit」;方案对比把「剪贴板+⌘V」列为推荐主方案,风险表 §7#5 隐含剪贴板快照/恢复是安全的。
- **实测是**:
  - **chunk≈20 数字 未验证**。loopback 谐波在同进程内 tap+post 造成死锁,只稳定观测到 chunk 0(keyDown+keyUp 都被 tap 看到)就 hang。需把 tap(监听)与 poster 拆成**两个进程**,再扫 chunk 1..200 × 帧间延迟 0..20ms 找可靠拐点。**不要把 20 当已定论硬编码**。
  - **端到端落字 未验证**(SPIKE 3 PARTIAL):consuming tap 只证明 CGEvent 被创建/post/进 HID 流,没证明进真实 App 文本框。verify agent 同样无法证明(loopback 吞事件 + Background session 读不到真实 App 字段)。**Method A vs Method B 目前无证据择一**,两者都留在开关后,直到两进程谐波或人工 TextEdit 测试定夺。
  - **剪贴板还原是有损的**(已证实):首次 `pbdump` 对一个含 5 种 flavor(HTML 3396B 等)的剪贴板返回 `types=[] string=nil`——那些是 **lazy/promised** pasteboard items,provider 不为本进程物化数据。**快照/恢复会静默丢掉 promised flavor**(浏览器/Office 常见)。`changeCount` 每次注入必 +2(360→361 写→362 恢复),剪贴板管理器会看到两次事件;`org.nspasteboard.ConcealedType/TransientType` 只是「约定」,守规矩的管理器才认。方案 A 恢复前需 ~400ms 让目标 App 异步读完剪贴板,过早恢复→粘贴空/旧。
- **应改成**:§2.2 把方案 A 的「必须保存并恢复用户剪贴板」标注为 **best-effort、对 promised data 有损**,并加决策:检测到多 flavor / promised item 就 warn 或跳过,或显式接受丢数据;§7#5 缓解措施同步加这条 caveat。「~20 UTF-16」改为「**待两进程谐波实测确定**」的开放项。Method B(CGEvent 打字法)天然不碰剪贴板,是绕开 promised-data 丢失的备选,保留。

### §2.4 权限链——**确认「ad-hoc 重建 Accessibility 授权会失效」,补 dev-loop 事实**
- **原 plan 说**:§2.4 第 2 点「开发期间每次重新编译签名变化,授权可能失效,需在辅助功能列表里删掉重加;正式版用稳定的 Developer ID 签名就不会」。
- **实测是**:**PLAN 这句是对的,且有量化证据**。ad-hoc 代码的 Designated Requirement 就是 cdhash(`codesign -d -r-` → `designated => cdhash H"..."`),whitespace-only 重建 cdhash 就变(`ec6e177a→83d10977`),所以每次重建 TCC 存的 grant 不再匹配 → Accessibility 授权不跨重建保留。Developer ID cert 会让 DR 变成 identity-based(identifier + anchor apple generic + TeamID),跨重建不变——这正是稳定 grant 的机制。
- **反证/降级**:SPIKE 3 补充一个 PLAN 未写的 **dev-loop 好消息**——如果开发时**直接从已授权的 Terminal 运行 ad-hoc 二进制**(而非 `open` .app),TCC 归因到 responsible process(Terminal),重签(CDHash 变)/改路径/改名都**保留** `AXIsProcessTrusted=true`。即:开发迭代用「从 Terminal 跑」零权限摩擦;只有打成独立 .app 用 `open` 启动时才吃 cdhash-per-build 的重授权。TCC.db 本身读不到(`authorization denied`,无 Full Disk Access),所以「grant 是否持久」只有 cdhash 机制这半可直接测,已测;弹窗/静默拒这半是**推断,未观测**。

### §3 OpenAI API——**确认 verbose_json 被拒;补非确定性 correctness bug + 4 条 refine**
- **原 plan 说**:§3 表格「gpt-4o-transcribe 仅支持 json/text,不支持 verbose_json/srt/vtt」;`language` 混输留空;prompt 做标点/去口头禅。
- **实测是**:
  - `verbose_json` **CONFIRMED 被拒** → `HTTP 400 unsupported_value`,`srt` 同样。只有 `json`/`text` 可用(→ **无 segment / 无词级时间戳 / 无 logprobs / 无 duration**,任何依赖时间或置信度的下游设计在本模型上无数据源)。
  - **PLAN 未写的 correctness bug**:同字节同参 5 次,**2 次返回繁体中文、3 次简体**(`那個幫我review...` vs `帮我review...`)。对中文语音输入是用户可见的数据损坏。加 prompt `使用简体中文输出` 后 4/4 简体。**该 clause 应列为 M5 强制项,不是可选打磨。** 且首 token 仍不稳(那个/你/你能/快,一次还改写成问句结尾 `?`)——**不要在首 token 上做任何精确串匹配 / diff / dictation-command 触发**。
  - `language=en` **没能**把混输强制成英文;`language=zh`/unset 结果一致,混输**留空最稳**(PLAN 已对,补证据)。`zzz` → `400 invalid_value`。
  - 25MB 上限**不是干净的 413/JSON**:服务器 mid-upload 掐 TLS,`curl (55) LibreSSL ... bad record mac` + `http_code 000` 空 body。**客户端必须本地预检文件大小**,不能只靠 HTTP status 分支。
  - json 响应含 PLAN 未提的 `usage` 对象(`total_tokens`/`input_token_details.text_tokens`/`audio_tokens`/`output_tokens`),可做实时成本核算。prompt 按 input text_tokens 计费(长 steering prompt 55 tokens,短音频上可能比音频还贵,**prompt 要短**)。
  - `gpt-4o-mini-transcribe` 也 200,是活的成本/降级杠杆,本 clip 上不明显更差。
  - 延迟实测:5.2s clip 稳态 0.6–1.0s,首次冷启 1.5s——UX 冷启单独预算。
  - 激进去口头禅 prompt 会误删真实内容(P2 把 `记得先跑一下test` 删成 `就是先跑一下test`)——prompt 清洗**保守**,verbatim 场景别用。
- **应改成**:§3「计划影响」补 `使用简体中文输出` 为强制 prompt clause(标定简繁);M5 验收把「简繁一致」纳入;§7 或 §2.3 加「上传前客户端预检 ≤25MB(TLS abort 非干净错误)」;response-shape spec 补 `usage` 对象;删掉任何依赖 segment/时间戳/置信度的下游假设。

### §6 分发:Developer ID + 公证——**今天不可执行,gate 在 $99/年账号后**
- **原 plan 说**:§6 完整给了 Developer ID 证书 → Archive → notarytool → stapler → spctl 流程,当作可执行步骤。
- **实测是**:`security find-identity -v -p codesigning` → **`0 valid identities found`**(任何证书都没有)。工具链在位:`notarytool --help` / `stapler` / `codesign` 都能跑;但 `notarytool submit` 需要 Apple Developer Program team(App Store Connect API key / app-specific password + Team ID),`codesign` 需要 Developer ID Application cert——**凭据全缺**。即 §6 是「工具就绪、账号阻塞」。
- **应改成**:§6 整节 gate 在「购买 Apple Developer Program($99/年)」之后,或改范围为 ad-hoc / 本机构建分发。**注意**:DP 钥匙串(§2.5)、稳定 TCC grant(§2.4)、公证分发(§6)**三个 blocker 塌缩成同一个 $99/年购买**,plan 应作为一个捆绑依赖、而非三个独立风险处理。

### §4 里程碑 M0——验收口径需改(见 §5 本文)
- **原 plan 说**:M0 内容含「Xcode 工程(§5)」,验收「App 启动只出现在菜单栏;热键按下控制台打点;⌘Q 干净退出」。
- **实测是**:M0 目标全部可达,但构建路径是 SwiftPM+bundle.sh 而非 Xcode 工程。菜单栏可见性由于 agent shell 在 Background launchd session(`launchctl managername=Background`)**无法自动截屏验证**,需 owner 肉眼看一次(见本文 §5)。
- **应改成**:M0「内容」把「Xcode 工程」换成「`Package.swift` + `bundle.sh`」;验收补一条「owner 在自己 Terminal 跑 `open dist/AIVoiceInput.app` 后,菜单栏右侧出现 mic 图标、Dock 无图标、无 TCC 弹窗」。

## 3. 完整可复现 build recipe(adversarial-verify 确认过的版本)

所有文件在 `.../scratchpad/spikes/build/`(MenuBarExtra)与 `.../scratchpad/spikes/hotkey/HotkeySpike/`(热键)。以下为逐字实测跑通版本。

### 3.1 `Package.swift`(SPIKE 1,`swift build -c release` → Build complete)
```swift

let package = Package(
    name: "AIVoiceInput",
    platforms: [
```

### 3.2 `Sources/AIVoiceInput/App.swift`(**文件名不要叫 `main.swift`**)
```swift
import SwiftUI

@main
struct AIVoiceInputApp: App {
    @State private var isRecording = false
    var body: some Scene {
        MenuBarExtra("AIVoiceInput", systemImage: "mic.fill") {
```

### 3.3 `bundle.sh`(构建+组装+ad-hoc 签名+校验;**不用 `--deep`**,已弃用)
```bash
#!/bin/bash
set -euo pipefail

APP_NAME="AIVoiceInput"
BUNDLE_ID="com.yujunzou.ai-voice-input"   # 与 PLAN 统一(spike 里是 com.apexnova.aivoiceinput,发布前改成这个)
VERSION="0.1.0"
CONFIG="release"

ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
BIN="$ROOT/.build/$CONFIG/$APP_NAME"
APP="$ROOT/dist/$APP_NAME.app"

swift build -c "$CONFIG" --package-path "$ROOT"

rm -rf "$APP"
mkdir -p "$APP/Contents/MacOS" "$APP/Contents/Resources"
cp "$BIN" "$APP/Contents/MacOS/$APP_NAME"

cat > "$APP/Contents/Info.plist" <<PLIST
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>CFBundleName</key>            <string>$APP_NAME</string>
    <key>CFBundleDisplayName</key>     <string>AI Voice Input</string>
    <key>CFBundleExecutable</key>      <string>$APP_NAME</string>
    <key>CFBundleIdentifier</key>      <string>$BUNDLE_ID</string>
    <key>CFBundlePackageType</key>     <string>APPL</string>
    <key>CFBundleShortVersionString</key><string>$VERSION</string>
    <key>CFBundleVersion</key>         <string>$VERSION</string>
    <key>CFBundleInfoDictionaryVersion</key><string>6.0</string>
    <key>LSMinimumSystemVersion</key>  <string>14.0</string>
    <key>LSUIElement</key>             <true/>
    <key>NSHighResolutionCapable</key> <true/>
    <key>NSMicrophoneUsageDescription</key>
    <string>AI Voice Input records your voice to transcribe it into text.</string>
</dict>
</plist>
PLIST

printf 'APPL????' > "$APP/Contents/PkgInfo"

# ad-hoc 直签整个 bundle(单可执行、无嵌套 framework,不需要 --deep)
codesign --force --sign - --identifier "$BUNDLE_ID" --timestamp=none "$APP"
codesign --verify --verbose "$APP"
echo "Built: $APP"
```

### 3.4 命令(逐条实测)

### 3.5 Keychain(去掉 DP flag,ad-hoc 跑通版本 — SPIKE 6)
```swift
let SERVICE = "com.yujunzou.ai-voice-input"
let ACCOUNT = "openai-api-key"
func baseQuery() -> [String: Any] {
    [ kSecClass as String: kSecClassGenericPassword,
      kSecAttrService as String: SERVICE,
      kSecAttrAccount as String: ACCOUNT ]
    // 不要设 kSecUseDataProtectionKeychain —— 那一行就是 -34018 的来源
}
func save(_ secret: String) -> OSStatus {
    SecItemDelete(baseQuery() as CFDictionary)                 // 幂等 upsert
    var q = baseQuery()
    q[kSecValueData as String] = Data(secret.utf8)
```
